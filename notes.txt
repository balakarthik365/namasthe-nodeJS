-----------------------**Run JS every where**---------------------------
Node JS is a cross platform, open-source javascript runtime built on chrome's v8 javascript engine.
Node JS helps to run javascript outside the web browser.
Node JS has event-driven architecture.
We can write JS at server side.
Capable of asynchronous I/O. also known as non-blocking I/O.

->Open JS foundation is a committee which works and maintains on Node JS.
->Ryan Dahl developed Node JS in 2009!!
------------------------------------------------------------------------
History of Node JS - 
In 2009, Node JS was developed by Ryan Dahl.
Where ever there is a JS there will be a JS engine.
Initially spider monkey(firefox) JS engine was used to write the Node JS. 
later, chrome's v8 was used to develop the Node JS.

Joyent - was company that trying to build similar kind of platform and offered Ryan to continue his work in their company and funded him.
Today, for the existence of Node JS Joyent has a big hand.

Earlier the name was Web JS, because this was meant to create web servers. Later, it was so powerful that not only create web servers but also other features.

Apache servers were blocking servers. Ryan want to build non-blocking servers so the focus was more for non-blocking.

In 2010, NPM was developed by Issac. NPM is a central registry of all packages that are used in the Node JS application.(Supported by only MacOS and Linux)

In 2011, it got windows support. making it much more accessible for bigger community.

In 2012, Ryan left the project. and was hand overed to the Issac(creator of NPM).

In 2014, Fedor a developer created a fork called io.js. now, joyent was maintaining Node JS and Fedor was maintaining io.js

In 2015, A Node JS foundation was formed and both io.js and Node JS was merged. The Node JS officially announced into the world.

In 2019, Node JS foundation and JS foundation merged to gether and formed a new committee as Open JS foundation.

If you wrap something inside the function it will be in its private space. Modules works similarly and can be accessed by module.exports or export default function.
IIFE - 
It Immediately Invoke the Function Execution.
It keeps the variables and functions safe, independent and private
when you require a module into other module the IIFE is executed

steps how require("path") works
1. Resolving the module. This will check if path is node_module/.js/.json
2. Loading the module. according to file type it loads the data
3. Compile the module. Wraps code inside an IIFE.
4. Code evaluation. module.exports happens
5. caching - Module that is returned is cached. Which means the code for require will run only once even when it is used in multiple files.

Any code is written using Node JS will be wrapped into IIFE and then passed into V8 engine.
