History of Node JS - 
In 2009, Node JS was developed by Ryan Dahl.
Where ever there is a JS there will be a JS engine.
Initially spider monkey(firefox) JS engine was used to write the Node JS. 
later, chrome's v8 was used to develop the Node JS.

Joyent - was company that trying to build similar kind of platform and offered Ryan to continue his work in their company and funded him.
Today, for the existence of Node JS Joyent has a big hand.

Earlier the name was Web JS, because this was meant to create web servers. Later, it was so powerful that not only create web servers but also other features.

Apache servers were blocking servers. Ryan want to build non-blocking servers so the focus was more for non-blocking.

In 2010, NPM was developed by Issac. NPM is a central registry of all packages that are used in the Node JS application.(Supported by only MacOS and Linux)

In 2011, it got windows support. making it much more accessible for bigger community.

In 2012, Ryan left the project. and was hand overed to the Issac(creator of NPM).

In 2014, Fedor a developer created a fork called io.js. now, joyent was maintaining Node JS and Fedor was maintaining io.js

In 2015, A Node JS foundation was formed and both io.js and Node JS was merged. The Node JS officially announced into the world.

In 2019, Node JS foundation and JS foundation merged to gether and formed a new committee as Open JS foundation.

Node JS has a event driven architecture capable of asynchronous I/O

JS -> Is a synchronous single threaded language. We need JS engine(V8, runs on a thread) to run the JS code. Runs synchronously!
Diff b/w synchronous and asynchronous:
Synchronous - synchronous code executes in a sequential manner, meaning that each operation must complete before the next one begins. 
              This can lead to blocking behavior, where the program waits for a task to finish before moving on to the next one.
Asynchronous - async code is non-blocking, it allows other code to run while waiting for an operation to complete. 
               It typically uses callbacks, promises, or async/await to handle the completion of tasks. 
               In contrast, synchronous code executes sequentially, blocking the execution of subsequent code until the current operation finishes.

JS is synchronous but in Node JS with power of Node it is asynchronous.

Node JS helps to do file operations, DB operations, HTTP network calls, timers and many more.

This operations are done by a library in Node JS. which is called libuv.

Node JS V8 -> libuv -> OS -> libuv -> v8 -> Node JS

v8 off loads to libuv whenever any OS, DB, HTTP, Timer operations happens.

libuv(A peace of C code) - it makes the asynchrounous I/O tasks made simple. It has event loop, thread pool and many queues.

Node JS has two major things V8 + libuv

v8 is main thread and libuv handles other os operations.

Google's V8 Engine:
2 type of languages - Interpretted and Compiled languages. 
                      In Interpreted language there is an interpretter which reads the code line by line. It executes fast initially.
                      In Compiled language. the code(high level language) is converted into Machine code then the machine code is executed. initially heavy but executed fast
JS - It is an other type of language is JS, It is not inpreted language or compiler language. It works with both compiler and interpretter.
Compilation is done by JIT(Just in time compilation). So JS has both compiler and interpretter.

When we run code what happens?
V8 Architecture
Parsing:
   1. Lexical analysis(Tokenization) happens - The code will be breaken down into lexical tokens. Code is read as token by token
         Ex: var a = 10; //here var, a, =, 10 are tokens.
   2. Syntax analysis - An abstract syntax tree(AST) is developed. This is known as parsing. It will throw an error when proper syntax is not written.
   3. AST - 
         The tokens are converted to An abstract syntax tree(AST)
   4. Interpretter(Ignition Interpretter) - 
         AST code is passed to interpretter(Ignition Interpretter - this will convert the AST into byte code) where code is read line by line.        
   5. Compiler(Turbofan Compiler) - crank shaft is the earlier compiler that is used. but later, removed in latest engines.
         If code has a repeated code lines, and if there is a chance to optimize that code. 
         This is called Hot code. 
         The Hot code is given to Turbofan compiler. this will convert this code into optimized machine code.
         Sometimes the compiler may also deoptimize the code. when compiler dont understand to optimize
   6. Execution - 
            Now, the byte code and optimized machine code is finally executed.
   7. Garbage collection(Orinoco, OilPan, Scavenger, MCompact) mark and sweep algorithm - 
         This happens parallelly to move unused code to garbage collection
 This whole thing is known as JIT compilation. Now, JS is super fast.
